## Architecture

- **Pattern:** Fase 1 — frontend-only SPA (Vue3 + Vite). Fase 2 — separated frontend SPA and backend REST API.
- **Data flow (fase 1):** Vue3 frontend only; schema data and app state loaded from and persisted to local files (export/import). The frontend calls an AI API service directly for mapping suggestions.
- **Data flow (fase 2):** Vue3 frontend → REST calls → .NET Core API → Entity Framework (persistence) + Claude API (AI suggestions).
- **AI integration (fase 1):** The frontend calls the Claude API (Anthropic) directly for three AI pillars: (1) field mapping suggestions with confidence scores via a two-layer approach (embedding-based candidate retrieval followed by LLM reasoning); (2) JSONata transformation expression suggestions for type/format-incompatible field pairs; (3) validation rule inference from OpenAPI field descriptions. Anthropic prompt caching is used to keep repeated calls against the same loaded spec fast. Schema compression (stripping non-essential fields) and an embedding index are built on schema load to handle large specs without exceeding context limits.
- **AI integration (fase 2):** AI calls are moved to the backend; the .NET Core API orchestrates them.

## Domain Model (code-level)

User-facing Dutch domain terms live in [VISION.md](VISION.md#begrensde-contexten). The list below is the code-level vocabulary — what types and modules are called inside the codebase. Add new terms here when introducing a concept.

| Term | Location | Description |
|---|---|---|
| `Schema` | `frontend/src/domain/schema.ts` | Parsed representation of one OpenAPI spec. Exposes fields through a queryable interface (`byId`, `has`, `all`, `roots`, `childrenOf`, `parentOf`, `pathOf`, `name`). Read-only after parse. Built via `parseOpenApiSchema(spec)`. |
| `SchemaField` | `frontend/src/types/schema.ts` | One field within a schema (`id`, `name`, `path`, `dataType`, `required`, `maxLength?`, `description?`). Hierarchy is **not** on the field — query it via `Schema` methods. |
| `SchemaFieldNode` | `frontend/src/domain/schema.ts` | Internal tree representation used during parsing only. Consumers must not traverse it. |
| `DataType` | `frontend/src/types/schema.ts` | `'string' \| 'number' \| 'boolean' \| 'date' \| 'object' \| 'array' \| 'unknown'` |
| `FieldMapping` | `frontend/src/types/mapping.ts` | One source-to-target field pairing plus its transformations. UI calls this a "koppeling"; code calls it a mapping. |
| `TransformationRule` | `frontend/src/types/mapping.ts` | One rule applied as part of a mapping. Kinds: `direct \| static \| expression \| truncate \| default \| cast \| date-format`. |
| Validation status | `frontend/src/utils/validationStatus.ts` | `'compatible' \| 'constrained' \| 'incompatible'` for a source/target field pair. |
| AI suggestion | `frontend/src/composables/useAISuggestions.ts` | OpenRouter-backed candidate mapping (source field → target field, confidence score). |

### Architectural seams

- **Domain seams live in `frontend/src/domain/`.** `Schema` is the first one. Consumers must not reach past it — for example, never traverse `SchemaField.children` (it does not exist on the public type).
- **Composables (`frontend/src/composables/`)** are the Vue-reactive wrappers around the domain. They own loading, error, and lifecycle.
- **Components (`frontend/src/components/`)** receive `Schema` references as props, not raw field arrays. Tree-aware queries go through `Schema` methods.

## Architecture Decision Records

| Decision | Rationale | Date |
|---|---|---|
| Vue3 + .NET Core Web API | Company standard stack; aligns with team expertise and future maintainability | 2026-04-16 |
| Fase 1: frontend-only (geen backend) | PoC fase vereist geen persistentie via backend; app state wordt opgeslagen en herladen via lokale bestandsexport. AI-integratie gebeurt direct vanuit de frontend. Backend en EF worden toegevoegd in fase 2. | 2026-04-16 |
| Claude API for AI suggestions | Strong structured-data reasoning; suitable for field-to-field mapping inference | 2026-04-16 |
| JSONata als transformatiestandaard | AI-suggesties genereren JSONata-expressies. JSONata is leesbaar, eval-baar in de browser (preview), en sluit aan op de AI-output zonder extra vertaallaag. | 2026-04-30 |
| `Schema` as domain seam, not `SchemaField[]` as prop shape | Components receive a `Schema` instance instead of a field array. Reason: top-level array iteration (`fields.length`, `fields.find`) missed nested fields, causing counter and lookup bugs. Tree traversal is now hidden behind `Schema` methods (`all`, `byId`, `childrenOf`, …). | 2026-05-11 |
